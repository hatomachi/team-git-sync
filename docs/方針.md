# Obsidian Git Custom Sync Plugin 開発要件と実装方針

## 1. プロジェクトの目的
ObsidianのVaultをチームで共有し、GitLab（Remote）と同期して運用するためのカスタムプラグインを開発する。既存のGitプラグインではカバーしきれない、チーム特有の競合解決ワークフロー（別名保存など）や、右ペインでの直感的な差分確認・個別Revert機能を実装し、Gitに不慣れなメンバーでも安全に同期できるDX（開発者体験）を実現する。

**【重要な前提条件：サブフォルダ単位のGit管理】**
Vaultディレクトリのルート自体はGit管理 **しない**。
Vault直下に `teamA`、`teamB` といったチーム別のサブフォルダが存在し、それぞれが独立したGitリポジトリとして管理され、GitLab上で権限分離されている構成を対象とする。そのため、プラグインは「Vaultルート」ではなく「現在アクティブなファイルが属するサブフォルダ（または設定で指定されたサブフォルダ）」を対象としてGit操作を行う必要がある。

## 2. 技術スタックと実装方針
ObsidianプラグインはNode.js環境（Electron）で動作するため、システムのGitを直接操作するアプローチを採用する。

* **ベース言語:** TypeScript (ObsidianプラグインAPI)
* **Git操作:** `simple-git` パッケージ
  * Node.jsの `child_process` をラップしたPromiseベースのライブラリ。複雑なGitコマンドの直列・分岐処理（Fetch -> Merge -> Conflict検知）を簡潔に書くために採用。
* **UI構築:** Obsidian標準のAPI (`ItemView`, `Modal`) ＋ Vanilla JS / DOM操作
  * 右ペインのリスト表示程度であればVanilla JSで十分だが、状態管理が複雑になる場合はSvelteの導入も検討する。
* **Diff表示:** `diff2html` パッケージ
  * `git diff` の出力を渡し、GitHubライクなSide-by-sideのHTMLを生成してモーダル内に描画する。
* **ファイル操作:** Node.js標準の `fs` / `path` モジュール ＋ Obsidian `Vault` API
  * 競合発生時の別名ファイルへの退避（リネーム・コピー）に使用。

## 3. 要件ごとの実装方針詳細

### A. 変更ファイルリストの表示（右ペイン）
* Obsidianの `ItemView` を拡張してカスタムペインを作成し、コマンドパレットから起動できるようにする。
* `simple-git` の `git.status()` を実行し、`modified`, `not_added`, `deleted` などの状態ごとにファイルをリスト化してDOMにレンダリングする。

### B. Remote Masterとの Visual Diff 表示
* リスト上のファイル名をクリックした際のイベントとして、Obsidianの `Modal` を開く。
* `git diff origin/master -- <filepath>` を実行して差分テキストを取得し、`diff2html` を用いてモーダル内に視覚的な差分UIをレンダリングする。

### C. 意図しない更新の個別 Revert
* リストの各ファイルアイテムにRevertボタンを配置する。
* クリック時に `git checkout origin/master -- <filepath>` または `git clean -fd <filepath>`（新規ファイルの場合）を実行し、状態を破棄する。実行後はリストを再フェッチしてUIを更新する。

### D. ワンクリック同期（Stage -> Commit -> Push）
* 右ペイン下部に「Sync」ボタンとコミットメッセージ用テキストボックス（デフォルト値：タイムスタンプ等）を配置。
* 正常系フロー：`git.add('./*')` -> `git.commit(message)` -> `git.push('origin', 'master')` を直列で実行する。

### E. 競合（Conflict）ハンドリングと別名退避
* Sync時の安全なフローとして、いきなりPushせず **Fetch -> Merge -> Push** の順で実行する。
* `git.merge(['origin/master'])` 実行時にコンフリクトエラーをキャッチする。
* `git ls-files -u` などで競合ファイルを特定する。
* 競合ファイルを Node.jsの `fs` を使って `filename_conflict_YYYYMMDD_HHMMSS.md` のようにリネームして退避させる。
* 元のファイルパスに対しては `git checkout --theirs <filepath>` (リモート優先) などで一旦Git上の競合状態をクリアし、退避した別名ファイルごと再度 `git add` -> `git commit` して `git push` を完了させる。

# 段階的な開発の段取り（フェーズ別タスク）

AIアシスタントへ：以下のフェーズ1から順番に実装を進めてください。各フェーズが完了し、動作確認が取れたら次のフェーズに進みます。

## Phase 1: プラグインの土台作りとGit連携テスト
1. Obsidianプラグインの標準的なボイラープレート（`main.ts`, `manifest.json`）を作成する。
2. `simple-git` をインストールし、プラグインのロード時に現在のVaultパスでGitリポジトリが初期化・認識されるか確認するロジックを実装する。
3. コマンドパレットから実行できるテストコマンドを作成し、`git status` の結果をコンソールに出力する。

## Phase 2: 右ペイン（カスタムビュー）とファイルリスト表示
1. `ItemView` を継承したカスタムビュークラスを作成し、Obsidianの右ペインに登録する。
2. カスタムビュー内に、`git status` で取得した「変更されたファイル」の一覧をレンダリングするUIを実装する。
3. リストを最新状態に更新するための「Refresh」ボタンを配置する。

## Phase 3: 個別Revert機能の実装
1. リストの各アイテムの横に「Revert（元に戻す）」アイコンボタンを追加する。
2. ボタンクリックで確認ダイアログ（Obsidianの `Notice` またはカスタムModal）を表示する。
3. 承認されたら、そのファイルに対して `git checkout -- <filepath>` を実行し、変更を破棄してビューをリフレッシュする。

## Phase 4: Visual Diff モーダルの実装
1. `diff2html` をインストールする。
2. リストのファイル名をクリックした際に発火するイベントを作成し、Obsidianの `Modal` を開く。
3. モーダルが開く際に `git diff origin/master -- <filepath>` を実行して差分テキストを取得し、`diff2html` でHTMLに変換してモーダル内に表示する。

## Phase 5: ワンクリック同期（正常系）の実装
1. 右ペインの下部にコミットメッセージ入力欄と「Sync」ボタンを追加する。
2. クリック時に `git.add('./*')` -> `git.commit()` -> `git.push()` を実行する非同期処理を実装する。
3. 処理中はボタンをローディング状態にし、完了時またはエラー時に Obsidian の `Notice` で通知を出す。

## Phase 6: 競合解決ロジック（別名退避）の実装（最重要）
1. Syncボタンの処理を修正し、`git.fetch()` -> `git.merge()` のフローを挟む。
2. Merge時にコンフリクトを検知する `try-catch` ブロックを実装する。
3. コンフリクト発生時、該当ファイルを特定し、`fs.rename` などを利用して `<元のファイル名>_conflict_<タイムスタンプ>.md` として退避させる。
4. Gitのインデックスを正常な状態に復帰させ（例：`git checkout --theirs` -> `git add`）、退避したファイルと一緒にCommit＆Pushを完了させる。
5. 「競合が発生したため、別ファイルとして退避して同期しました」という通知をユーザーに出す。
